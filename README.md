# BD Lab 3

Pet-проект на Django для учета парковки: водителей, автомобилей, гаражей, парковочных мест, пропускных карт и журнала посещений.

Проект читает данные из существующей PostgreSQL-базы и показывает их в веб-интерфейсе. Для части страниц доступны выгрузки в CSV, а также сводные отчеты по посещениям.

## Что умеет проект

- список водителей, машин, гаражей, парковочных мест, пропускных карт и посещений;
- агрегированные отчеты по водителям, машинам, гаражам и годам;
- экспорт таблиц и отчетов в CSV;
- работа поверх уже созданной базы данных без миграции схемы через Django.

## Стек

- Python 3.x
- Django 4.2
- PostgreSQL
- HTML templates

## Как запустить

1. Создайте и активируйте виртуальное окружение.
2. Установите зависимости:

```bash
pip install django psycopg2-binary
```

3. Проверьте настройки подключения к базе в [bd_lab_3/settings.py](bd_lab_3/settings.py).
4. Убедитесь, что PostgreSQL-база `garage_full` существует и в ней уже есть таблицы.
5. Запустите сервер:

```bash
python manage.py runserver
```

После этого приложение будет доступно по адресу `http://127.0.0.1:8000/`.

## Маршруты

- `/` — главная страница;
- `/drivers/` — водители;
- `/cars/` — автомобили;
- `/garages/` — гаражи;
- `/parking/` — парковочные места;
- `/cards/` — пропускные карты;
- `/visits/` — журнал посещений;
- `/visits_by_year/` — сводка по годам;
- `/drivers/csv/`, `/cars/csv/`, `/garages/csv/`, `/parking/csv/`, `/cards/csv/`, `/visits/csv/`, `/visits_by_year/csv/` — CSV-выгрузки.

## Структура

- `bd_lab_3/` — настройки проекта, корневые URL и конфигурация Django;
- `posts/` — приложение с моделями, представлениями, сервисными запросами и шаблонами;
- `posts/services.py` — агрегирующие запросы для отчетов;
- `posts/utils.py` — общая CSV-выгрузка;
- `posts/templates/` — HTML-страницы.

## Идеи для улучшения

- вынести параметры подключения к базе в переменные окружения и убрать пароль из `settings.py`;
- исправить дублирование функций в `posts/views.py` и привести имена к одному стилю;
- добавить `requirements.txt` или `pyproject.toml`, чтобы проект было проще запускать;
- оформить таблицы через общий base template и добавить навигацию между разделами;
- перевести часть логики отчетов на более явные сервисы или class-based views;
- добавить тесты для CSV-экспорта и агрегирующих запросов;
- убрать `managed = False`, если схема должна поддерживаться самим Django, либо отдельно описать схему БД в README;
- добавить фильтры по датам, водителям и гаражам, чтобы отчеты были полезнее как pet project.

## Примечание

Сейчас проект лучше всего воспринимать как учебную заготовку. Чтобы сделать из него сильный pet project, стоит добавить аккуратный UI, немного бизнес-логики и более безопасную конфигурацию окружения.
