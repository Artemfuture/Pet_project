# BD Lab 3

Учебный pet-проект на Django для учета парковочной инфраструктуры. Приложение предоставляет веб-интерфейс для работы с существующей базой данных PostgreSQL, позволяя просматривать информацию о водителях, автомобилях, гаражах, парковочных местах, пропускных картах и журнале посещений, а также формировать сводные отчеты и экспортировать данные в CSV.

## Возможности

- Просмотр списка водителей;
- Просмотр автомобилей;
- Просмотр гаражей;
- Просмотр парковочных мест;
- Просмотр пропускных карт;
- Просмотр журнала посещений;
- Формирование агрегированных отчетов по водителям, автомобилям, гаражам и годам;
- Экспорт таблиц и отчетов в формате CSV.

## Стек

- Python 3
- Django 4.2
- PostgreSQL
- Django ORM
- HTML Templates

## Архитектура проекта

Проект разделен на несколько логических уровней:

- **Models** — работа с существующей схемой PostgreSQL через Django ORM (`managed = False`);
- **Views** — обработка HTTP-запросов и отображение страниц;
- **Services** — агрегирующие запросы и бизнес-логика формирования отчетов;
- **Utils** — общая логика экспорта данных в CSV;
- **Templates** — HTML-шаблоны пользовательского интерфейса.

## Запуск проекта

### 1. Клонировать репозиторий

```bash
git clone <repository_url>
cd bd_lab_3
```

### 2. Создать виртуальное окружение

```bash
python -m venv venv
```

Активировать:

**Windows**

```bash
venv\Scripts\activate
```

**Linux / macOS**

```bash
source venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install django psycopg2-binary
```

### 4. Настроить подключение к PostgreSQL

Укажите параметры подключения к базе данных в файле:

```text
bd_lab_3/settings.py
```

Проект использует существующую базу данных **garage_full**, поэтому перед запуском убедитесь, что она создана и содержит необходимые таблицы.

### 5. Запустить сервер

```bash
python manage.py runserver
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000/
```

## Основные маршруты

| URL | Описание |
|------|----------|
| `/` | Главная страница |
| `/drivers/` | Водители |
| `/cars/` | Автомобили |
| `/garages/` | Гаражи |
| `/parking/` | Парковочные места |
| `/cards/` | Пропускные карты |
| `/visits/` | Журнал посещений |
| `/visits_by_year/` | Сводный отчет по годам |

### Экспорт в CSV

Для большинства разделов доступна выгрузка данных в формате CSV:

- `/drivers/csv/`
- `/cars/csv/`
- `/garages/csv/`
- `/parking/csv/`
- `/cards/csv/`
- `/visits/csv/`
- `/visits_by_year/csv/`

## Особенности проекта

- Работа с существующей PostgreSQL-базой без управления схемой через миграции Django (`managed = False`);
- Использование сервисного слоя для формирования агрегированных отчетов;
- Повторное использование общей логики экспорта данных в CSV;
- Разделение бизнес-логики и представлений для повышения читаемости кода.
